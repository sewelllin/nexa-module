WHY NEXA

不是再造一个 IoT 平台，
而是让每台设备拥有自己的 Agent。

传统 IoT 平台把设备变成云端的被动终端。Nexa Module 把意图理解作为云端认知能力，把实时执行权、安全边界和业务连接留在设备端。

NEVER LOSE THE DEVICE

设备可发现、可重连、
可追踪、可恢复。

“固定 IPv6”提供稳定寻址，完整的在线保障体系负责处理网络切换、断线、服务故障与设备异常。任何单一机制都不是全部答案。

01

身份与地址双绑定

固定 IPv6、device_id 与设备证书三者绑定。地址用于寻址，证书用于确认“它确实是这台设备”。

02

长连接与心跳

业务、AI 两条连接分别保活；记录 last_seen、网络类型、信号质量和连接代次。

03

自动重连与网络切换

指数退避、抖动重试、会话恢复；Flex 版可在 4G 与 Wi‑Fi 间切换，BLE 提供近场救援入口。

04

多入口与多区域

主备域名、双可用区意图服务、备用接入地址，避免单个服务入口导致设备集体离线。

05

离线继续工作

本地白名单 Skill、最近策略缓存、队列持久化与 store-and-forward，恢复连接后补传。

06

失联告警与恢复

分级超时、设备失联告警、远程诊断、看门狗自恢复和安全归属重建，形成闭环。

网络异常→本地继续执行安全任务→切换链路 / 自动重连→恢复会话并补传→平台更新在线状态

NEXA LINK PROTOCOL

两条 TCP + TLS 直连通道，
一套可恢复的设备协议。

01

可靠分帧

4-byte 长度前缀 + version + message_type + payload，为业务链路和 AI 链路统一解决粘包与拆包。

02

双向身份

设备证书或预共享密钥，TLS 1.3 加密；device_id 不作为唯一信任依据。

03

幂等与防重放

request_id、timestamp、nonce、sequence 与签名保证命令只执行一次。

04

离线降级

任一服务器不可达时独立重连；本地开放白名单 Skill，缓存事件并在会话恢复后补传。

CONTROL PLANE WITHOUT DATA RELAY

保留最小控制面，
但不让它成为业务中转节点。

IPv6 地址管理

设备固定地址用于稳定寻址。地址、设备证书与租户归属建立映射，支持吊销、迁移与重新绑定。

身份和权限

双向 TLS 确认通信双方。ACL、Skill 权限、风险等级与命令有效期共同限制设备能执行什么。

生命周期服务

保留证书签发、OTA、版本与可观测性。这些能力管理设备，但不代理设备与应用服务器之间的实时业务数据。

可用性边界

工程目标是快速发现和恢复失联。对于断电、SIM 停机或物理损坏，系统会告警并保留最后状态，而不是宣称网络永不失效。
