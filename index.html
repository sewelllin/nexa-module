Product Thesis

不是再造一个 IoT 平台，而是让每台设备拥有自己的 Agent。

01

自然语言优先

串口文字、本地屏幕、语音转文字、云端远程控制，都统一进入 Natural Language Input。

02

云端只做意图匹配

意图服务器接收自然语言、设备状态和 Skill Manifest，返回结构化 Skill Plan。

03

端侧拥有执行主权

设备本地验证 skill、参数、权限和风险等级，最终动作永远由设备 Agent 执行。

Architecture

端侧 Agent + 云端意图服务器 + 应用服务器直连。

Input Adapter

接入串口、屏幕、App、云端远程输入，并统一标准化为自然语言对象。

Skill Registry

维护设备本地能力清单，向意图服务器提供可匹配、可验证的能力描述。

Plan Validator

校验 AI 返回计划，确保参数合法、权限满足、风险可控。

TCP Network Manager

设备、意图服务器和应用服务器之间统一使用 TCP 传输，优先 IPv6 直连。

DAIP Protocol

所有控制输入都是自然语言，所有执行动作都是结构化 Skill Plan。

设备请求

{
  "type": "INTENT_RESOLVE_REQ",
  "input": "把继电器 1 打开 5 秒",
  "source": "local_serial",
  "skills": ["relay.control", "timer.schedule"],
  "state": {"relay_1": "off"}
}

意图响应

{
  "type": "INTENT_RESOLVE_RESP",
  "intent": "relay.pulse",
  "confidence": 0.97,
  "plan": [
    {"skill_id": "relay.control", "arguments": {"channel": 1, "state": "on"}},
    {"skill_id": "timer.schedule", "arguments": {"delay_seconds": 5}}
  ]
}

Security Model

AI 可以建议，设备必须裁决。

设备身份唯一设备 ID、证书或预共享密钥
加密传输TCP + TLS 1.3 或轻量加密层
防重放时间戳、nonce、请求签名、幂等 ID
本地策略权限、风险等级、高危操作二次确认

Use Cases

从开发板 Demo 到行业设备，都走同一套 Agent 链路。

智能硬件：灯光、风扇、门锁、家电、语音设备。

工业控制：继电器、传感器、Modbus、CAN、工单系统。

智慧农业：温室、灌溉、补光、环境采集、告警联动。

开发者模组：GPIO、UART、HTTP、定时任务和日志上报。
