WHY NEXA

不是再造一个 IoT 平台，
而是让每台设备拥有自己的 Agent。

传统 IoT 平台把设备变成云端的被动终端。Nexa Module 把意图理解作为云端认知能力，把实时执行权、安全边界和业务连接留在设备端。

01

自然语言即 API

文字、串口文本、语音转写和远程控制统一为自然语言消息，不再为每项操作设计专用界面。

02

云端只做认知

意图服务器结合自然语言、设备状态与 Skill Manifest，返回结构化、可解释的执行计划。

03

端侧拥有主权

本地 Agent 校验能力、参数、权限和风险。AI 可以建议，设备必须裁决。

NEXA LINK PROTOCOL

TCP 是传输底座，Nexa Link 定义可靠边界。

01

可靠分帧

4-byte 长度前缀 + version + message_type + payload，解决 TCP 粘包与拆包。

02

双向身份

设备证书或预共享密钥，TLS 1.3 加密；device_id 不作为唯一信任依据。

03

幂等与防重放

request_id、timestamp、nonce、sequence 与签名保证命令只执行一次。

04

离线降级

云端不可达时仅开放白名单本地 Skill；缓存事件，恢复后补传。

KEY PRODUCT DECISIONS

对原始设想，建议这样定型。

输入放在哪里？

自然语言先到设备，再由设备请求意图服务器。设备可统一接收本地与远程输入，并在上传前做来源认证、隐私过滤和上下文裁剪。

IPv6 固定地址够吗？

地址解决可达性，不解决信任。必须使用设备证书、双向 TLS、访问控制列表和密钥轮换。

完全不要平台吗？

去中心化执行，但保留最小控制面。量产仍需要身份签发、OTA、版本与可观测性；它们不参与实时业务转发。

所有通信只用 TCP？

可以，但要明确 TCP + 应用层协议。定义分帧、心跳、重连、超时、幂等、流控与错误码；生产环境必须加 TLS。
